# Портал капитанов

Веб-портал для капитанов десяток бизнес-сообщества «Аномалия».

- 📚 **Библиотека** — база знаний со статьями и записями мероприятий, поиск и фильтры.
- ✍️ **Форма обращений** — капитаны присылают кейсы на разбор, вопросы, предложения выступлений.
- 🛠 **Админка** — дашборд с метриками, очередь запросов, CRUD статей / мероприятий / спикеров, аннотации и whitelist.

Доступ только для верифицированных капитанов (whitelist по Telegram-нику).

---

## Стек

- Vanilla HTML/CSS/JS, без фреймворков
- Tailwind CSS через CDN + кастомный `css/styles.css`
- Google Fonts: Oswald (заголовки) + Mulish (тело)
- [Supabase](https://supabase.com) PostgreSQL + Auth bypass через anon key + RLS
- marked.js + DOMPurify (оба через CDN) — markdown-рендер
- Хостинг: Vercel (статика, автодеплой из `main`)

## Первый запуск

### 1. Supabase

1. Создай проект на [supabase.com](https://supabase.com) (free tier).
2. В **Project Settings → API** скопируй `Project URL` и **publishable/anon** ключ.
3. Пропиши их в `js/supabase-config.js`:
   ```js
   const SUPABASE_URL      = 'https://xxxxx.supabase.co';
   const SUPABASE_ANON_KEY = 'sb_publishable_...';
   ```
4. Открой **SQL Editor → New query**, скопируй всё содержимое `supabase/migrations/0001_init.sql` и нажми **Run**.
5. Проверь, что всё создалось:
   ```sql
   SELECT table_name FROM information_schema.tables WHERE table_schema = 'public';
   -- 8 таблиц: allowed_users, annotations, articles, categories, events, likes, requests, speakers
   SELECT name FROM categories ORDER BY sort_order;
   -- 5 категорий
   SELECT tg_handle FROM allowed_users;
   -- должен быть стартовый @my_metodolog (или владелец, которого ты там прописала)
   ```

### 2. Админ-пароль

В `js/admin-config.js`:
```js
window.ADMIN_PASSWORD = 'твой-пароль';
```
Защита MVP: пароль ложится на фронтенд, поэтому считай его «не случайно зайдёт», а не «невзламываемой». При росте — вынести CRUD-операции в Vercel Serverless Functions с `service_role` ключом и заменить RLS-политики.

### 3. Деплой на Vercel

1. Подключи репозиторий в Vercel (Import project).
2. В **Framework Preset** выбери **Other** (это статика, билд не нужен).
3. Root directory — корень репозитория.
4. Деплой. Каждый push в `main` — автодеплой.

`vercel.json` уже лежит в репе: статика + кеш-заголовки для `css/js/img` и `favicon.svg`.

## Как пользоваться (для Мэри)

### Добавить капитана

1. Зайди в `/admin.html`, введи пароль.
2. В шапке → **Whitelist**.
3. Слева — «Добавить одного» (один ник) или «Добавить списком» (textarea, каждый ник с новой строки).
4. `@` добавляется автоматически, дубли игнорируются.

### Завести статью

1. **Статьи** → «Новая статья».
2. Заполни заголовок, выбери категорию, теги через запятую, статус = «Опубликовано».
3. Markdown — в поле «Тело». На десктопе справа live-превью, на мобайле — табы «Текст / Превью».
4. Сохранить. Статья появится в библиотеке сразу.

### Завести мероприятие

1. (Если нужен новый) **Спикеры** → «Новый спикер» → сохранить.
2. **Мероприятия** → «Новое мероприятие».
3. URL видео: YouTube (`watch?v=…` / `youtu.be/…` / `shorts/…`), Kinescope (`kinescope.io/…`), RuTube (`rutube.ru/video/…`) — платформа определяется автоматически.
4. «Материалы» — любое количество ссылок, «+ Добавить ссылку» / «Удалить».

### Обработать запрос

1. **Запросы** — в шапке рядом с «Запросы» бейдж с количеством новых.
2. По умолчанию фильтр «Новый». Кликни «Открыть» на строке.
3. В модалке: все поля запроса + «Мои пометки» + селект статуса.
4. При переключении в **«Отвечен»** или **«Закрыт»** первый раз — автоматически проставится `answered_at` / `closed_at`.

### Прочитать обратную связь

**Аннотации** — бейдж со счётчиком в шапке. Длинный текст сворачивается до 4 строк с «Показать полностью». Обработала — «Удалить», бейдж уменьшится.

### Поменять пароль админки

Отредактируй `window.ADMIN_PASSWORD` в `js/admin-config.js` → push в `main` → автодеплой.

## Известные упрощения MVP

- **Админский CRUD идёт через anon key + открытые RLS-политики** — защищено только паролем на фронте. Для продакшна — вынести в Vercel Serverless Functions с `service_role` ключом.
- Лайк-счётчики денормализованы (`articles.likes`, `events.likes`) + инкрементятся двумя запросами с клиента — при одновременном нажатии от двух пользователей возможна гонка. На MVP-объёмах несущественно; при росте — сделать атомарный RPC.
- Поиск в библиотеке работает на клиенте (один раз грузим весь опубликованный контент). Быстро и достаточно для сотен статей. Если станет тысячи — переводить на серверный поиск (Supabase `tsvector` или edge function).
- `service_role` ключ в коде не используется и не коммитится — нужен только если захотим перенести админские операции в serverless.

## Troubleshooting

**«Доступ только для верифицированных капитанов» при корректном нике**
Проверь в Supabase таблицу `allowed_users` — сравни регистр, пробелы и лишние символы (`@ivanov` ≠ `ivanov ` с пробелом).

**Счётчик просмотров не растёт**
RPC `increment_views` вызывается при открытии статьи. Проверь в Supabase: **Database → Functions** должна быть функция `increment_views`. Если нет — перезапусти миграцию.

**Видео не отображается**
Автодетект поддерживает YouTube / Kinescope / RuTube. Другие платформы покажут ссылку «Смотреть видео». Если ссылка корректная, но не играет — проверь консоль браузера на X-Frame-Options / CSP от платформы.

**Бейджи новых запросов / аннотаций не обновляются**
Счётчики читаются один раз при загрузке страницы админки. Переоткрой страницу, чтобы цифры пересчитались.
